Math Context Menu Example

📋 How to use the Math Context Menu

The Math context menu provides powerful operations for mathematical expressions. To access it:

  1. Draw a circle or rectangle around the expression you want to work with (this selects it)
  2. Choose an operation from the menu
  • Computation: Write √16=, draw around it, Compute numerical result
  • Variable substitution: Write x=3, then 2x+5=, draw around the second expression, Compute numerical result (will use x=3 value)
  • Function plot: Write f(x)=x², draw around it, Evaluate function → Set range from -5 to 5
  • Reversible function plot: Write y=x², draw around it, Evaluate function → you can choose to plot y = f(x) or x = f(y)
  • Complex expression: Write a=2, then 3a²+2a-1=, draw around the expression, Edit variables → Change a value, Compute numerical result (will use the updated value)
  • Check diagnostic (both tasks): Write y=x²+2x, draw around it, Check diagnostic → See both "Numerical Computation" and "Function Evaluation" diagnostics
  • Division by zero: Write 1/0=, draw around it, Check diagnostic → See error message about division by zero

The Check diagnostic option analyzes your mathematical expression and provides comprehensive feedback for two different tasks:

🔢 Numerical Computation

Checks if the expression can be computed to get a numerical result (e.g., 2+2=, √16=).

📊 Function Evaluation

Checks if the expression can be evaluated and plotted as a function (e.g., f(x)=x², y=2x+1).

Diagnostic Results:

Each task will show one of these diagnostic codes:

  • ✓ ALLOWED: The task can be performed successfully
  • ⚠ INCOMPLETE_EXPRESSION: The expression needs to be completed
  • ⚠ TOO_MANY_VARIABLES: Too many undefined variables
  • ✗ DIVISION_BY_ZERO: Computation leads to division by zero
  • ✗ NUMBER_OVERFLOW: Result is too large
  • ✗ INVALID_OPERATION: Invalid mathematical operation
  • ℹ NO_VARIABLE: No undefined variables to evaluate
  • ℹ NOT_APPLICABLE: Task doesn't apply to this expression

💡 Pro tip: Use the diagnostic to understand why a task might fail and which operations are possible for your expression before attempting them.

  • End expressions with = or ? to indicate you want a result
  • Variables are automatically detected when you define them (e.g., x=5)
  • You can use the same variable in multiple expressions
  • The solver supports common math functions: sin, cos, tan, sqrt, exp, log, etc.
  • For evaluation, the function must have a clear input-output relationship (e.g., y=f(x))
  • Use "Check diagnostic" to verify if an expression is ready for both computation and evaluation before attempting either task
  • The diagnostic feature shows results for two tasks: Numerical Computation (🔢) and Function Evaluation (📊)
  • Green diagnostic (✓ ALLOWED) means you can safely perform that specific task
  • Each task can have a different diagnostic result - for example, an expression might be evaluable but not computable
