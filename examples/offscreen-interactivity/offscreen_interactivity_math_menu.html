Math Menu Example

📋 Two menus to explore

This example demonstrates the two math-specific menus:

  • Context menu — draw a circle or rectangle around an expression to select it, then choose an operation
  • Math action menu (∑) — click the ≡ menu icon in the toolbar, then select Math (∑) to access global settings and variable management

Context menu

  • Computation: Write √16=, draw around it → Compute numerical result
  • Variable substitution: Write x=3, then 2x+5=, draw around the second expression → Compute (uses x=3)
  • Edit variable & recompute: Draw around 2x+5= → Set variable value, change x, then Compute again
  • Function plot: Write f(x)=x², draw around it → Evaluate function, set range -5 to 5
  • Check diagnostic: Write y=x²+2x, draw around it → Check diagnostic
  • Division by zero: Write 1/0=, draw around it → Check diagnostic

∑ Action menu — global variables

  • Override a block variable globally: Write x=3 then 2x=, draw around 2x= and compute → result is 6. Then open ≡ → Math (∑) → Edit Variables → add global x = 10 → Apply. Compute again → result is 20 (global x overrides block x=3).
  • Shared constant: Open ≡ → Math (∑) → Edit Variables → add global A = 42. Write 2A= and compute — A is resolved globally.

Click the ≡ in the toolbar, then select Math (∑). Settings apply globally to the whole canvas.

Show Block Overlays (∑)

Toggles visual overlays on recognized math blocks — shows block boundaries, computed results, and status indicators.

Draw result as strokes

When enabled, computed results are drawn as ink strokes next to the expression instead of displayed as an overlay panel.

Show Dependencies on Hover

Hovering over a math block highlights its variable sources (green) and dependent blocks (orange) with dependency arrows.

Highlight on Select

When a block is selected, related blocks are highlighted and unrelated ones are dimmed.

Edit Variables

Opens the variable definitions editor showing all variables known by the solver:

  • BLOCK — defined by a math expression (e.g. x=3). Read-only here.
  • API — set programmatically per block. Editable.
  • API_GLOBAL — set globally (no block scope). Editable. Overrides BLOCK definitions.
  • UNDEFINED — referenced but not yet defined. Editable — set a value to resolve it.

Add Global Variable: Use the "Add Global Variable" section at the bottom to create a new API_GLOBAL variable by entering a name and a value. It will be available to all expressions on the canvas.

The Check diagnostic option (context menu) analyzes your expression and provides feedback for two tasks:

🔢 Numerical Computation

Checks if the expression can be computed to get a numerical result (e.g., 2+2=, √16=).

📊 Function Evaluation

Checks if the expression can be evaluated and plotted as a function (e.g., f(x)=x², y=2x+1).

Diagnostic Results:

  • ✓ ALLOWED: The task can be performed successfully
  • ⚠ INCOMPLETE_EXPRESSION: The expression needs to be completed
  • ⚠ TOO_MANY_VARIABLES: Too many undefined variables
  • ✗ DIVISION_BY_ZERO: Computation leads to division by zero
  • ✗ NUMBER_OVERFLOW: Result is too large
  • ✗ INVALID_OPERATION: Invalid mathematical operation
  • ℹ NO_VARIABLE: No undefined variables to evaluate
  • ℹ NOT_APPLICABLE: Task doesn't apply to this expression

💡 Pro tip: Use the diagnostic to understand why a task might fail before attempting it.

  • End expressions with = or ? to indicate you want a result
  • Variables are automatically detected when you define them (e.g., x=5)
  • You can use the same variable in multiple expressions
  • The solver supports common math functions: sin, cos, tan, sqrt, exp, log, etc.
  • For evaluation, the function must have a clear input-output relationship (e.g., y=f(x))
  • Global variables (API_GLOBAL) override block-defined variables of the same name
  • Use "Check diagnostic" to verify if an expression is ready before computing
